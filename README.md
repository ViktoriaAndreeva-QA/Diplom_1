# Проект по тестированию класса Burger

## Описание проекта
Данный проект содержит юнит-тесты для класса `Burger` из приложения Stellar Burgers. 
Класс `Burger` отвечает за создание бургера, добавление и удаление ингредиентов, 
перемещение ингредиентов, расчет итоговой стоимости и формирование чека.

## Технологии
- Python 3.14
- pytest 9.0.2
- pytest-cov 7.0.0
- unittest.mock

## Фикстуры (conftest.py)
- `mock_bun` — создает мок булочки с заданными параметрами
- `mock_ingredient` — создает мок ингредиента (соус/начинка)
- `burger_with_ingredients` — создает бургер с предустановленными ингредиентами

## Покрытие тестами
Класс `Burger` покрыт тестами на **100%**.

## Тестовые сценарии

### Тесты для метода `set_buns`
- ✅ Выбор булочки
- ✅ Замена одной булочки на другую

### Тесты для метода `add_ingredient`
- ✅ Добавление одного ингредиента (соус или начинка)
- ✅ Добавление соуса и начинки вместе
- ✅ Добавление нескольких одинаковых ингредиентов
- ✅ Добавление нескольких разных начинок

### Тесты для метода `remove_ingredient`
- ✅ Удаление ингредиента по индексу
- ✅ Удаление всех ингредиентов
- ✅ Удаление единственного ингредиента
- ❌ Ошибка при удалении из пустого бургера
- ❌ Ошибка при индексе меньше минимального
- ❌ Ошибка при индексе больше максимального
- ❌ Ошибка при очень большом индексе

### Тесты для метода `move_ingredient`
- ✅ Перемещение ингредиента на другую позицию
- ✅ Обратимость перемещения (возврат к исходному порядку)
- ❌ Ошибка при индексе меньше минимального
- ❌ Ошибка при индексе больше максимального
- ❌ Ошибка при перемещении в пустом бургере

### Тесты для метода `get_price`
- ✅ Расчет цены только с булочкой
- ✅ Расчет цены с булочкой и одним ингредиентом
- ✅ Расчет цены с булочкой и несколькими ингредиентами
- ❌ Ошибка при отсутствии булочки

### Тесты для метода `get_receipt`
- ✅ Формирование чека только с булочкой
- ✅ Формирование чека с булочкой и одним ингредиентом
- ✅ Формирование чека с разными комбинациями ингредиентов
- ✅ Проверка формата чека
- ✅ Проверка регистра типа ингредиентов (нижний регистр)
- ❌ Ошибка при отсутствии булочки

## Использование моков
В тестах для методов `get_price` и `get_receipt` используются моки:
- `mock_bun` — для имитации булочки с заданной ценой и названием
- `mock_ingredient` — для имитации ингредиентов с заданными параметрами

## Параметризация
Активно используется параметризация для проверки множества сценариев с минимальным дублированием кода:
- Разные типы булочек
- Разные комбинации ингредиентов
- Разные индексы для удаления и перемещения
